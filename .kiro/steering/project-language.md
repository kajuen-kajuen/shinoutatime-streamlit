# プロジェクト言語設定

## 基本方針

本プロジェクト「しのうたタイム」では、**日本語を標準言語**とします。

## 適用範囲

### 日本語で記述すべきもの

1. **ドキュメント**
   - README.md
   - 要件定義書（requirements.md）
   - 設計書（design.md）
   - 実装タスクリスト（tasks.md）
   - その他すべてのマークダウンドキュメント

2. **コードコメント**
   - Pythonファイル内のコメント
   - docstring（関数やクラスの説明）
   - インラインコメント

3. **ユーザー向けメッセージ**
   - Streamlitアプリケーション内の表示テキスト
   - エラーメッセージ
   - 警告メッセージ
   - 情報メッセージ

4. **Kiroとのコミュニケーション**
   - Kiroへの質問や指示
   - Kiroからの返答
   - コードレビューのコメント
   - タスクの説明

### 英語で記述すべきもの

1. **プログラミング要素**
   - 変数名
   - 関数名
   - クラス名
   - モジュール名
   - ファイル名（Pythonファイル、設定ファイルなど）

2. **技術用語**
   - 必要に応じて英語の技術用語を使用可能
   - ただし、日本語での説明を併記することが望ましい

## Kiroへの指示

- すべての返答は日本語で行ってください
- コードを生成する際、コメントとdocstringは日本語で記述してください
- ドキュメントを作成・更新する際は日本語で記述してください
- エラーメッセージや説明は日本語で提供してください
- 技術的な内容であっても、可能な限り日本語で説明してください

## 例外

以下の場合は英語の使用を許可します：

1. 外部ライブラリやフレームワークの公式ドキュメントを引用する場合
2. エラーメッセージがシステムから自動生成される場合
3. 国際的な標準規格や仕様を参照する場合
4. GitHubのissueやPull Requestで英語圏の開発者とコミュニケーションする場合

## 開発環境に関する方針

### Pythonコマンドの動作確認

本プロジェクトでは、**Pythonコマンドの動作確認を不要**とします。

#### 適用範囲

- Pythonスクリプトの実行確認
- テストコードの実行確認
- パッケージのインストール確認
- 開発サーバーの起動確認

#### 理由

開発環境の制約により、Pythonコマンドの実行が困難な場合があります。そのため、以下の方針で進めます：

- コードの静的解析（構文チェック、型チェック）を優先
- ロジックの正確性をコードレビューで確認
- 実際の動作確認はユーザー側で実施
- テストコードは記述するが、実行結果の確認は不要

#### Kiroへの指示

- Pythonコマンドを実行しないでください
- `python`、`pytest`、`streamlit run`などのコマンド実行を避けてください
- コードの正確性は静的な分析とレビューで確認してください
- テストコードは記述しますが、実行は求めないでください

## 理由

本プロジェクトは日本のVTuberファンを対象とした日本語サイトであり、開発チームも日本語話者が中心です。日本語を標準言語とすることで、以下のメリットがあります：

- コミュニケーションの円滑化
- ドキュメントの理解しやすさ向上
- メンテナンス性の向上
- 新規参加者の参入障壁の低減
