## バージョン管理の方針

### バージョン番号の形式

本プロジェクトは [セマンティック バージョニング 2.0.0](https://semver.org/lang/ja/) に従います。

バージョン番号は `MAJOR.MINOR.PATCH` の形式で表記されます：

- **MAJOR**: 互換性のない大きな変更を行った場合に増加
- **MINOR**: 後方互換性のある機能追加を行った場合に増加
- **PATCH**: 後方互換性のあるバグ修正を行った場合に増加

### 変更の分類

各バージョンの変更は、以下のカテゴリに分類されます：

- **Added（追加）**: 新機能の追加
- **Changed（変更）**: 既存機能の変更
- **Deprecated（非推奨）**: 近い将来削除される機能
- **Removed（削除）**: 削除された機能
- **Fixed（修正）**: バグ修正
- **Security（セキュリティ）**: セキュリティに関する変更

### リリースの種類

- **正式リリース**: `1.0.0`, `1.1.0`, `2.0.0` など
- **ベータ版**: `1.0.0-beta.1`, `1.0.0-beta.2` など
- **リリース候補版**: `1.0.0-rc.1`, `1.0.0-rc.2` など

## 変更履歴の記録方法

新しいバージョンをリリースする際は、以下のテンプレートを使用してください：

```markdown
## [バージョン番号] - YYYY-MM-DD

### Added
- 追加された新機能の説明

### Changed
- 変更された既存機能の説明

### Deprecated
- 非推奨となった機能の説明

### Removed
- 削除された機能の説明

### Fixed
- 修正されたバグの説明

### Security
- セキュリティに関する変更の説明
```

### 記録のガイドライン

1. **日付形式**: ISO 8601形式（YYYY-MM-DD）を使用
2. **変更の説明**: 簡潔かつ明確に記述
3. **ユーザー視点**: ユーザーにとって意味のある変更を記録
4. **リンク**: 関連するIssueやPull Requestへのリンクを含める（該当する場合）
5. **順序**: 最新のバージョンを上部に配置
6. **Unreleased**: 次のリリースに含まれる予定の変更を記録

### 例

```markdown
## [1.1.0] - 2024-03-15

### Added
- プレイリスト作成機能を追加 (#123)
- お気に入り機能を追加 (#124)

### Changed
- 検索結果の表示速度を改善 (#125)
- UIデザインを更新 (#126)

### Fixed
- 日付ソートの不具合を修正 (#127)
- タイムスタンプ変換のエラーを修正 (#128)
```
